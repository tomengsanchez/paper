Backend stack
Custom PHP MVC framework, not Laravel/CodeIgniter.
Core framework lives under Core/ (router, base controller, DB, auth, CSRF, logger, mail, migrations).
Requests enter through public/index.php (front controller) and are routed via Core\Router to App\Controllers\... methods.
Frontend stack
Server-rendered PHP views, no SPA framework.
Uses Bootstrap 5, jQuery, and Select2 (with a Bootstrap 5 theme) for UI and enhanced form controls.
Main layout supports:
- Sidebar or top-navigation layouts (per-user via UserUiSettings).
- Optional “super mobile-friendly” mode, which on small screens:
  - Stacks Bootstrap grid columns vertically.
  - Makes tables more compact with horizontal scrolling.
  - Turns the sidebar into an overlay drawer with a hamburger toggle and backdrop.
Application structure
App\Controllers\ – HTTP and API controllers (e.g. dashboard, profile, structure, grievance, users, settings, security, notifications, etc.). – PDO-based models, no heavy ORM; mostly one model per table/domain.
App\Views\ – module-based PHP templates with a shared layout (layout/main.php) and partials (pagination, toolbars, sidebars).
Clear MVC pattern with role/capability-based access checks in controllers and menus.
Export: CSV export from Profile, Structure, Grievance, Users, and Library. Select Columns and Export dialogs show all available fields (ListConfig::getExportColumns). CsvExporter streams CSV; Grievance::listForExport resolves lookup names.
Database layer & schema
DB config via config/database.php, used by Core\Database (PDO singleton).
Schema defined by PHP migration scripts under database/migration_*.php and seeders under database/seed_*.php. Migrations: run via php cli/migrate.php; status via --status; rollback via --rollback or --rollback --steps=N (LIFO, requires callable 'down' in each migration).
Main domains: users/roles/capabilities, projects, profiles, structures, grievances + grievance status/history, app/user settings, notifications, audit log, API tokens.
Security-related schema additions:
- app_settings: stores security and UI configuration (2FA, login throttling, password policy, email settings, UI preferences, etc.).
- users.password_changed_at: used for password expiry checks.
- user_password_history: per-user history of password hashes for enforcing “last N passwords” reuse rules.
